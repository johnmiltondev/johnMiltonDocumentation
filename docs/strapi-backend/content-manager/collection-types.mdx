# Collection types

Los collection types es el primer apartado de ***Content Manager***.
Son basicamente las tablas de la base de datos que contienen toda la información acerca de:

- Usuarios
- Productos
- Compras
- Subscripciones al boletín
- Direcciones de usuarios
- etc.

</div>

## Antes de empezar

Para crear un nuevo item en cualquiera de las tablas solo es necesario presionar el boton de ***Create new entry*** que esta en la esquina superior derecha.

<div style={{textAlign: 'center'}}>
  <img src="/img/docs/strapi/cm-create-new-entry.png" alt="strapi create new entry button" />
</div>

Antes de comenzar existen algunos puntos importante que deben seguirse sin importar la tabla:

- Al crear un item en cualquier tabla aparecera 2 botones
  - Publish: este publica la información y la hace visible para los usuarios de la web
  - Save: este boton salva la información como un borrados y los usuarios de la we no podran verlos
    - Esto sirve para guardar información que aún no se ha autorizado publicar pero se salva para adenlantar y poder revisar el trabajo antes de publicar.

<div style={{textAlign: 'center'}}>
  <img src="/img/docs/strapi/cm-publish-save.png" alt="strapi create new entry button" />
</div>

- Todos los campos obligatorios (asterisco rojo) deben ser llenados para poder publicar los datos.
- Todos los items que contengan ***relaciones*** deberan tambien ser llenados, esto es clave para que la información se publique correctamente. Estas relaciones aparecen así como en la siguiente imagen.

<div style={{textAlign: 'center'}}>
  <img src="/img/docs/strapi/cm-relation.png" alt="strapi create new entry button" />
</div>

- Los audios, volumenes, liveEvents son los diferentes 3 tipos de productos que la pagina tiene. Cuando se cree alguno de ellos siempre sera necesario crear tambien un producto en la tabla de productos ya que esa tabla es la que tiene la información de venta.

- El campo ***category*** es necesario para el uso de los filtros. Es necesario usar el valor dependiedo del tipo de producto tiene los siguientes campos:
  - audio: para audios y volumenes
  - hipnosis-online: para liveEvents que sean de tipo hipnosis-online
  - mental-evolution: para liveEvents que sean de tipo mental-evolution
  - mental-reset: para liveEvents que sean de tipo mental-reset
  - tour: para liveEvents que sean de tipo tour

Con lo anterior entendido podemos comenzar a explicar cada tabla.

## Product

Son todos los productos que aparecen en la web en las secciones donde aparecen con tarjetas de productos.

Algunos de los campos mas importante de llenar son:

- El campo de ***type*** se usa en la tabla de Productos para saber que tipo de producto es, es necesario ingresar el dato:
  - audio
  - liveEvent
  - volumen

- Para audios y volúmenes es importante ingresar el dato de expiration days, ya que este se usa
para calcular la fecha de expiración del producto. Solo ingresar el número.
  - 30 para audios
  - 120 para volúmenes

- Las relaciones que aparecen abajo son la manera de conectar a un producto al tipo de producto que es. Esto hace que todos los datos especificos de ese producto se jalen.
  - audioProduct: solo llenar este campo si el producto que se va a conectar es un audio.
  - volume_product: solo llenar este campo si el producto que se va a conectar es un volúmen.
  - liveEventProduct: solo llenar este campo si el producto que se va a conectar es un evento en vivo.

## Purchases

La tabla de todas las compras registradas que hicieron los usuarios, incluyendo los audios gratis.

<div style={{textAlign: 'center'}}>

Se pueden crear compras si se desea resolver problemas a usuarios que no tienen sus productos habilitados o si por otro lado se desea regalar algun producto. Para regalar un productos es importante que el usuario al que se le regalara un producto ya haya creado una cuenta en la pagina, ya que se necesita para poder hacer esto.

:::note
Importante llenar el valor de ***active*** a ***true*** para que le aparezca al usuario el producto.
Igual llenar la fecha de compra y de expiración.
:::

## Volume

Cosas importantes para ingresar un Volúmen.

1. Colocar todos los audios que este volúmen tiene para que puedan conectarse. Los audios debieron haberse creado anteriormente antes de poder ingresarlos en este campo.
2. La categoria debe ser "audio"
3. Hacer la relación del producto. Para esto tambien tiene que estar creado el producto para que de esta forma puedan salir en la pagina web los datos del volúmen.
4. Llenar todos los datos con el asterisco rojo
